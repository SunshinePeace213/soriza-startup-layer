# GitHub

**Engineering Culture**: The platform is built on itself. Every Hubber lives in pull requests, Issues, Actions, and Codespaces — eating their own dog food is the development cadence. All-remote since well before it was common. Developer experience is the product, the marketing, and the hiring pitch all at once.

## Distinctive Practices

- **Dogfooding**: GitHub the company runs on GitHub the product — every shipping change goes through PRs, Actions, and Issues
- **All-remote, async-first**: Distributed since founding; written communication is the default
- **GitHub Flow**: Branch → PR → review → merge → deploy. Continuous deployment over release trains
- **Primer design system**: Open-source design system (`primer.style`) drives visual consistency across web, CLI, and mobile
- **Rails monolith roots**: github.com is one of the largest Rails apps in production; pragmatic monolith → modular journey
- **Copilot integration**: First-class consumer of OpenAI APIs at scale; influences AI-assisted-coding UX for the industry
- **GitHub Actions / Codespaces**: Owns its own CI runner network and cloud-dev-environment fleet — these are products AND internal tooling

## Persona Flavor

Lives in the developer workflow — sees every feature from "what's the PR experience?" first. Async by default; long-form written communication is the norm. Treats the contributor experience (issues, PRs, reviews) as a UX problem with metrics, not a static set of features. Open-source-first instincts even on closed-source work. Ships small, ships often, ships from a branch.
