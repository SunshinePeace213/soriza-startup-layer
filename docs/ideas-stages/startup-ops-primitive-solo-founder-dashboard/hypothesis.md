# Hypothesis — Solo-Founder Operations Primitive

**Candidate ID:** cand-startup-ops-primitive-solo-founder-dashboard  
**Status:** Testable ✓ (Gate 1 — Advance)

---

## Sharpened Hypothesis Sentence

**Solo founders who manage multiple disconnected SaaS tools (Notion, Linear, Stripe, Beehiiv) spend 3–5 hours weekly on context-switching and manual data aggregation, yet lack a unified operations dashboard they can customize themselves without engineering resources. This forces them to either tolerate the context-switching tax or commission developer time to rebuild their dashboard each time operational priorities shift.**

---

## Dimensions (Provisional)

### WHO
**Solo founders and indie hackers operating micro-SaaS or bootstrapped ventures, paying for multiple SaaS tools (Notion, Linear, Stripe, Beehiiv, etc.), primarily English-language market.** The founder (Ringo) is literally this customer — uses multiple tools, pays the context-switching tax directly.

**Specificity:** Named role (solo founders, indie hackers) + domain context (managing multiple SaaS tools) + segment (bootstrapped, English-language) + self-identified user ✓

---

### HOW OFTEN
**3–5 hours per week spent on context-switching and manual data aggregation across tools.**

This is derived from the problem's language ("meaningful time weekly") and is testable via time audits or founder self-report.

**Specificity:** Concrete frequency (weekly) + magnitude (3–5 hours) ✓

---

### HOW SEVERELY
**Lost operational hours weekly (time cost) + inability to self-customize dashboard without technical co-founder or hired developer, creating a dependency that slows operational pivots.**

Pain modes:
1. Direct cost: 3–5 hours/week × founder's effective rate = meaningful revenue drag for a bootstrapped founder.
2. Indirect cost: When operational priority shifts (e.g., focus moves to churn this week, onboarding next), the founder cannot reshape their dashboard without a developer, delaying decision-making or forcing workarounds.

**Specificity:** Dual cost model (time + inability to adapt) + magnitude (hours per week) + context (bootstrapped founders) ✓

---

### STATUS QUO
**Solo founders manually aggregate data across 5+ disconnected tools (Notion for tasks, Linear for bugs, Stripe for revenue, Beehiiv for content, etc.). They either:**
- **Tolerate context-switching:** lose the hours, live with fragmented views, miss cross-tool insights.
- **Commission workarounds:** build custom Zapier/Make flows or hire a developer to build a dashboard, neither of which adapts as priorities change.

No existing tool allows rapid, self-service dashboard customization at this scale for micro-founders.

**Specificity:** Named current state (manual aggregation across named tools) + explicit trade-offs (tolerate cost or commission dev time) ✓

---

## Falsifiability Checklist

This hypothesis is falsifiable via:

1. **WHO**: Interview solo founders about their tool stack → confirm 5+ active tools, confirm manual aggregation.
2. **HOW OFTEN**: Time audit or founder self-report → confirm 3–5 hours/week (or refute).
3. **HOW SEVERELY**: 
   - Survey willingness-to-pay for a self-customizable unified dashboard.
   - Measure friction: How many times per month does a founder need to re-request a dashboard change? How long is the dev cycle?
   - Ask directly: "Would a tool that let you reshape your dashboard in 5 minutes, without code, save you time?" and measure conviction.
4. **STATUS QUO**: Audit the customer's current tool ecosystem and workflow — confirm they're building workarounds (Zapier, Make, custom dev) or abandoning cross-tool insights.
