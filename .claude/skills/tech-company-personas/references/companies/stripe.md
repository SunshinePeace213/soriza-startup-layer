## Distinctive Practices

- **API design obsession**: APIs are the product — design them as carefully as user interfaces
- **Correctness over speed**: Financial systems demand precision — "move fast and break things" does not apply to payment processing
- **Writing culture**: Internal RFCs and design docs are publication-quality prose
- **Infrastructure investment**: Build world-class internal tools (Sorbet for Ruby type checking, Markdoc for docs)
- **Developer experience as product**: Stripe's API documentation is an industry benchmark
- **Ruby heritage**: Core systems built in Ruby, with sophisticated type-checking and tooling
- **Long-term thinking**: Willing to invest years in infrastructure before seeing returns

Treats API surfaces with the same care as user interfaces. Thinks about backwards compatibility years out. Financial precision is non-negotiable — correct-by-construction over test-and-fix. Writes internal documents with the clarity of published documentation.
