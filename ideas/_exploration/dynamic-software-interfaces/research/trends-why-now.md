# Trends / Why-Now — Findings

## Key findings

- **AI coding-agent revenue has crossed a threshold that confirms mass-market willingness to pay.** Cursor went from $100M ARR (Jan 2025) → $500M (Jun 2025) → $1B (Nov 2025) → $2B (Feb 2026), doubling in three months — the fastest B2B SaaS scale on record. Lovable hit $20M ARR in two months (fastest in European startup history); Bolt.new hit $40M ARR in six months. This pace signals that the tooling layer enabling user-modifiable software is already generating real revenue and has crossed from hobbyist to mainstream. — [TechCrunch / Cursor $2B ARR](https://techcrunch.com/2026/03/02/cursor-has-reportedly-surpassed-2b-in-annualized-revenue/), [NxCode / Lovable & Bolt ARR](https://www.nxcode.io/resources/news/v0-vs-bolt-vs-lovable-ai-app-builder-comparison-2025)

- **The "primitives + user modification" pattern is now shipping production infrastructure.** shadcn/ui (100k+ GitHub stars, adopted by Vercel/Linear/Supabase) and Radix Primitives (130M+ monthly npm downloads) have made "copy-paste raw components that users style and extend" the de facto React standard. This directly enables a "ship primitives, let users compose the interface" product model — the infrastructure exists and is already widely deployed. — [ZenrioTech / shadcn & Radix headless architecture](https://zenriotech.com/blog/shadcn-radix-ui-headless-component-architectures-standard)

- **Ink & Switch published the definitive "Malleable Software" manifesto in June 2025**, co-authored by Geoffrey Litt (now at Notion), arguing for shared-data workspaces where users reshape tools with minimal friction. Their prototypes — Patchwork (web-based collab canvas where code is stored as data and shared via URL), Potluck (dynamic plaintext notes), Embark (structured travel planner) — demonstrate LLM-assisted tool creation "in just a few minutes." The essay is the intellectual anchor that legitimises this space to investors and builders. — [Ink & Switch — Malleable Software essay](https://www.inkandswitch.com/essay/malleable-software/)

- **Vercel's AI SDK (20M+ monthly downloads) codified "generative UI" as a first-class pattern**: LLMs stream React Server Components directly, assembling interface at runtime rather than at build time. A 2025 Google paper found users preferred AI-generated HTML/CSS/JS over markdown 83% of the time, calling UI generation an "emergent capability" requiring no UI-specific training. This means generating bespoke interfaces on demand is now tractable, not research-grade. — [Vercel AI SDK — Generative UI docs](https://ai-sdk.dev/docs/ai-sdk-ui/generative-user-interfaces), [Vercel blog AI SDK 3.0](https://vercel.com/blog/ai-sdk-3-generative-ui)

- **MCP (Model Context Protocol) shipped "MCP Apps" in late 2025 — tools that return interactive UI components rendered directly in the conversation.** By January 2026 the spec was official; the ecosystem hit 97M monthly SDK downloads and 10,000+ community servers. MCP acts as a protocol layer for "ship a server/primitives, AI assembles UI for each user's context" — exactly the infrastructure backbone the thesis requires. — [MCP Apps official announcement Jan 2026](https://blog.modelcontextprotocol.io/posts/2026-01-26-mcp-apps/), [MCP one-year review Nov 2025](https://blog.modelcontextprotocol.io/posts/2025-11-25-first-mcp-anniversary/)

- **Figma Make (launched May 2025, GA July 2025) lets anyone prompt a Figma design into a live interactive app**, including Supabase backend integration. Figma's 4M+ designer user base now has a path from static design to dynamic, modifiable app without writing code. This signals the design tool category has accepted "interface as a prompt-driven, malleable artifact" as a core product direction, not an experiment. — [Figma Make GA announcement](https://www.figma.com/blog/figma-make-general-availability/), [CMSWire on Figma Make beta](https://www.cmswire.com/digital-experience/figma-make-beta-launches-to-turn-designs-into-apps-and-prototypes/)

- **App store submission glut confirms the "build cost collapse" is already here and creating a new problem.** Apple's App Store saw 557,000 new app submissions in 2025 — a 24% YoY jump, the highest since 2016 — driven by AI-assisted vibe coding. 70% of store visitors discover via search; 90% never pass the 10th result. The collapse in build cost has shifted the constraint from making software to distributing and personalising it — directly the gap the malleable-interface thesis addresses. — [CEOWORLD — vibe coding app store flood](https://ceoworld.biz/2026/03/11/vibe-coding-is-flooding-the-app-stores-with-new-apps-standing-out-just-became-the-hardest-part/)

- **AI coding tools are reducing software production cost by 6–13× on specific workloads** (custom CRM: $500K → $80K; test suite: $200K → $15K), with 55% faster task completion in controlled trials. At this economics, "a personal tool built for one user or one team" becomes financially viable for the first time — the precondition for the "software for one" model the thesis envisions. — [Lunatech — AI is collapsing the cost of software (May 2026)](https://blog.lunatech.com/posts/2026-05-12-ai-is-collapsing-the-cost-of-software)

- **Claude Artifacts introduced a shareable, forkable, AI-powered runtime in 2025**: users can build interactive apps inside Claude, share them, and recipients get their own copy to modify — no API keys, no hosting cost. This is the first mainstream consumer-grade proof-of-concept for "software you can fork and reshape like a document." Usage scales against each user's own Claude subscription, making the distribution model essentially free for creators. — [Anthropic — Claude-powered artifacts](https://www.anthropic.com/news/claude-powered-artifacts), [Claude Artifacts help center](https://support.claude.com/en/articles/9487310-what-are-artifacts-and-how-do-i-use-them)

- **40% of new SaaS MVPs in 2026 will be built primarily via AI app builders; 25% of YC W2025 startups ran on 95%-AI-generated codebases.** This normalises AI-assembled software as a delivery mechanism, lowering the psychological barrier for end-users and businesses to accept (and eventually demand) interfaces that are themselves generated and customisable rather than fixed. — [daily.dev — vibe coding 2026](https://daily.dev/blog/vibe-coding-how-ai-changing-developers-code/)
