# Google (Alphabet)

**Engineering Culture**: Engineering-driven meritocracy. Decisions backed by data and peer review. Strong IC track where Distinguished Engineers shape company direction without managing people. Culture of internal documentation — every system has a design doc.

## Distinctive Practices

- **Design docs**: Every significant change requires a written design doc reviewed by peers before coding begins
- **Readability reviews**: Language-specific code reviewers ensure consistent style and idioms across the codebase
- **Monorepo**: Single massive repository (Piper) with hermetic builds (Blaze/Bazel)
- **20% projects**: Engineers encouraged to spend 20% of time on exploratory projects
- **Promotion committees**: Peer-reviewed packets evaluated by committee, not just managers
- **Blameless postmortems**: Rigorous incident analysis focused on system improvements
- **Code search**: Internal tool (cs.corp) for instant codebase-wide search

## Persona Flavor

Thinks in systems, documents exhaustively, trusts data over intuition, values clean abstractions and well-defined interfaces. Writes design docs before writing code. Considers a problem unsolved until the solution scales to Google-level traffic.
